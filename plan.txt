Это план работы над статьей.

В статье будем рассматривать приёмы коллективной работы над некоторым 
многофайловым проектом. В качестве примера будем разрабатывать комплект
документации для эксплуатации чайника.

Статья будет в виде пьесы с тремя действующими лицами. 

Первый - организатор работ.
Второй - автор комплекта документации, предлагающий оформление проекта в 
формате MarkDown.
Третий - ещё один автор, предлагающий форматирование документации в формате
LaTeX.

Два формата нужны для того, чтобы рассмотреть работу с ветками.
Изменения в проект, который изначально будет предложен вторым действующим
лицом будут вноситься PullRequest'ами.
